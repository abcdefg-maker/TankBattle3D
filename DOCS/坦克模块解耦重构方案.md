# 坦克模块解耦重构方案

> **版本**: v1.0
> **日期**: 2026-03-04
> **目标**: 将坦克相关代码（模型、武器、AI、特效）从各文件中剥离，整合为自包含的独立模块

---

## 一、当前问题：耦合分析

### 1.1 坦克相关逻辑分布现状

| 文件 | 坦克相关内容 | 问题 |
|------|-------------|------|
| `js/tank.js` | 坦克模型、移动、炮塔、后坐力 | 纯坦克类，但缺乏自主射击能力 |
| `js/bullet.js` | 子弹模型、飞行、拖尾 | 独立文件，但由 game.js 外部调度 |
| `js/ai.js` | 敌方AI决策+生成器 | 与坦克紧耦合，应归属坦克模块 |
| `js/effects.js` | 坦克专属特效 + 通用特效混合 | `createTankHitEffect`、`createHitFlash` 等与坦克绑定 |
| `js/collision.js` | 子弹碰撞检测 | 直接调用 effectsManager（7处）和 audioManager（6处） |
| `js/game.js` | 射击触发、坦克创建/销毁、AI调度 | 38+ 处直接操作坦克，是最大的耦合源 |

### 1.2 关键耦合点

```
game.js ──── 直接创建 Tank 实例（4处）
         ├── 直接调用 bulletManager.shoot()（2处）
         ├── 直接调用 effectsManager.createMuzzleFlash()（2处）
         ├── 直接调用 tank.startRecoil()（2处）
         ├── 直接访问 tank.group.visible（2处）
         └── 直接管理 enemyTanks[] 和 enemyAIs[]

collision.js ── 在碰撞检测内直接触发特效（7处）
             └── 在碰撞检测内直接播放音效（6处）
```

**核心问题**：`game.js` 既是调度者又是执行者，坦克没有自主能力。

## 三、各模块详细设计

### 3.1 Tank.js — 坦克实体

**来源**: 当前 `js/tank.js` 的全部内容

**职责**: 坦克3D模型构建、位置/旋转控制、后坐力动画、护盾/加速状态、碰撞盒

**对外接口**:
```javascript
class Tank {
    constructor(scene, options)

    // 位置与移动
    setPosition(x, z)
    getPosition() → Vector3
    getRotation() → number
    calcMovePosition(forward, deltaTime) → Vector3
    applyPosition(pos)
    rotate(direction, deltaTime)
    setVisible(visible)     // 新增：替代直接访问 group.visible

    // 炮塔
    setTurretRotation(angle)
    getTurretWorldAngle() → number  // 新增：获取炮塔世界角度
    getBarrelTipPosition() → Vector3
    getShootDirection() → Vector3

    // 后坐力
    startRecoil()
    updateRecoil(deltaTime)

    // 状态
    canShoot() → boolean
    resetShootCooldown()
    takeDamage(amount) → boolean
    activateShield(duration)
    activateSpeedBoost(duration)
    updateBoundingBox()

    // 生命周期
    update(deltaTime)
    destroy()
}
```

**改动点**:
- 新增 `setVisible(visible)` 方法，封装 `group.visible` 访问
- 新增 `getTurretWorldAngle()` 方法
- 其余保持不变，仅从 `js/tank.js` 迁移到 `js/tank/Tank.js`

---

### 3.2 TankWeapon.js — 武器系统

**来源**: 从 `game.js` 的射击逻辑 + `bullet.js` 的 BulletManager 中提取

**职责**: 管理坦克的射击行为，包括创建子弹、炮口闪光、后坐力触发

**对外接口**:
```javascript
class TankWeapon {
    constructor(scene, effectsManager, audioManager)

    // 射击（内部处理：创建子弹+炮口闪光+后坐力+音效）
    fire(tank) → Bullet | null   // 如果 canShoot 为 false 则返回 null

    // 子弹管理
    update(deltaTime)
    getBullets() → Bullet[]
    removeBullet(bullet)
    clear()
}
```

**fire(tank) 内部流程**:
```
1. 检查 tank.canShoot() → 否则返回 null
2. tank.resetShootCooldown()
3. tank.startRecoil()
4. pos = tank.getBarrelTipPosition()
5. dir = tank.getShootDirection()
6. bullet = new TankBullet(scene, pos, dir, tank.type)
7. effectsManager.createMuzzleFlash(pos)
8. audioManager.playShoot()
9. return bullet
```

**效果**: `game.js` 中的射击逻辑从 6 行代码缩减为 `weapon.fire(tank)`。

```javascript
class TankBullet {

```

**改动**: 仅改文件名和路径，类内容不变。

### 3.4 TankEffects.js — 坦克专属特效

**来源**: 从 `effects.js` 中提取坦克相关方法

**职责**: 处理坦克被击中、被击杀时的视觉反馈

**对外接口**:
```javascript
class TankEffects {
    constructor(effectsManager)   // 依赖通用特效管理器

    onTankHit(tank, position)      // 击中：爆炸+火星+碎片+闪烁
    onTankDestroyed(position)      // 击杀：大爆炸+浓烟+大量火花
}
```

**内部实现**: 组合调用 `effectsManager` 的通用方法（`createExplosion`、`createSmoke`、`createSparks`、`createDebris`）+ 坦克专属的闪烁逻辑。

**effects.js 保留内容**:
- `createExplosion()` — 通用爆炸
- `createSmoke()` — 通用烟雾
- `createSparks()` — 通用火花
- `createDebris()` — 通用碎屑
- `createMuzzleFlash()` — 炮口闪光
- `createBrickHitEffect()` — 砖墙击中
- `createSteelHitEffect()` — 钢墙击中
- `createBaseHitEffect()` — 基地击中
- `update()` / `clear()` — 粒子生命周期管理

**从 effects.js 移除**:
- `createHitFlash(tank)` → 移至 TankEffects
- `createTankHitEffect()` → 移至 TankEffects
- `createTankDestroyEffect()` → 移至 TankEffects

### 3.7 TankManager.js — 坦克总管理器（核心）

**来源**: 从 `game.js` 中提取坦克管理逻辑

**职责**: 统一管理所有坦克实例（玩家+敌人），对外提供高层接口

**对外接口**:
```javascript
class TankManager {
    constructor(scene, effectsManager, audioManager, collisionSystem)

    // 玩家坦克
    createPlayerTank(options, spawnPos) → Tank
    getPlayerTank() → Tank

    // 敌人管理
    initSpawner(levelData)
    updateEnemies(deltaTime, baseMeshes)
    getEnemyTanks() → Tank[]
    getAliveEnemyCount() → number
    getRemainingEnemyCount() → number

    // 射击（统一入口）
    playerShoot() → boolean       // 玩家射击，返回是否成功

    // 碰撞处理（接收碰撞结果，触发对应特效）
    handleBulletCollision(result)  // 根据碰撞类型触发坦克特效和音效

    // 子弹
    getBullets() → Bullet[]
    updateBullets(deltaTime)

    // 全局
    getAllTanks() → Tank[]
    update(deltaTime)
    clear()

    // 事件回调（由 game.js 设置）
    onEnemyKilled: (enemy, deathPos) => void
    onPlayerKilled: () => void
    onPlayerHit: () => void
}
```

## 四、collision.js 重构 — 纯碰撞检测

### 4.1 改造目标

将 `checkBulletCollisions` 改为**只返回碰撞结果**，不触发任何特效或音效。

### 4.2 新接口

```javascript
// 改造前：内部调用 effectsManager 和 audioManager
checkBulletCollisions(bullet, playerTank, enemyTanks, effectsManager, audioManager, game)

// 改造后：纯检测，只返回碰撞信息
checkBulletCollisions(bullet, playerTank, enemyTanks)
    → { type, position, target? } | null
```

### 4.3 返回值示例

```javascript
{ type: 'brick',         position: Vector3 }
{ type: 'steel',         position: Vector3 }
{ type: 'base',          position: Vector3 }
{ type: 'enemy_hit',     position: Vector3, target: enemyTank }
{ type: 'enemy_killed',  position: Vector3, target: enemyTank }
{ type: 'player_hit',    position: Vector3, target: playerTank }
{ type: 'player_killed', position: Vector3, target: playerTank }
{ type: 'boundary' }
```

### 4.4 特效触发转移

碰撞结果由 `TankManager.handleBulletCollision(result)` 处理：
- `brick` → `effectsManager.createBrickHitEffect(pos)`
- `steel` → `effectsManager.createSteelHitEffect(pos)`
- `base` → `effectsManager.createBaseHitEffect(pos)`
- `enemy_hit` → `tankEffects.onTankHit(target, pos)`
- `enemy_killed` → `tankEffects.onTankDestroyed(pos)`
- `player_hit` → `tankEffects.onTankHit(target, pos)`
- `player_killed` → `tankEffects.onTankDestroyed(pos)`

音效同理由 TankManager 调用 audioManager。

## 五、game.js 重构 — 变为高层调度器

### 5.1 改造前后对比

| 职责 | 改造前（game.js 直接做） | 改造后（委托给谁） |
|------|------------------------|--------------------|
| 创建玩家坦克 | `new Tank(scene, opts)` | `tankManager.createPlayerTank(opts, pos)` |
| 创建敌人坦克 | `new Tank()` + `enemyAIs.push()` | `tankManager.initSpawner()` + 自动管理 |
| 玩家射击 | 6 行代码 | `tankManager.playerShoot()` |
| 敌人AI+射击 | 30+ 行代码 | `tankManager.updateEnemies(dt)` |
| 子弹更新 | 遍历+碰撞+switch | `tankManager.updateBullets(dt)` |
| 碰撞特效 | collision 内部触发 | `tankManager.handleBulletCollision(result)` |
| 击杀计分 | switch case 内 | `onEnemyKilled` 回调 |
| 清理实体 | 逐个销毁 | `tankManager.clear()` |

### 5.2 game.js 精简后的核心结构

```javascript
class Game {
    init() {
        // ... 场景/渲染器/光照 ...
        this.tankManager = new TankManager(scene, effectsManager, audioManager, collisionSystem);
        this.tankManager.onEnemyKilled = (enemy, pos) => { this.score += ...; };
        this.tankManager.onPlayerKilled = () => { this.onPlayerDeath(); };
    }

    updatePlayerInput(deltaTime) {
        const tank = this.tankManager.getPlayerTank();
        // 移动/旋转/炮塔控制（保留在 game.js，因为涉及输入和相机）
        // 射击
        if (this.keys['Space']) {
            this.tankManager.playerShoot();
        }
    }

    updateEnemies(deltaTime) {
        this.tankManager.updateEnemies(deltaTime, this.gameMap.baseMeshes);
    }

    updateBullets(deltaTime) {
        this.tankManager.updateBullets(deltaTime);
    }
}
```
