# TankBattle3DD - 野战场景设计策划

## 一、设计目标

将当前平坦的网格地图改造为**开放式野战场景**：
- 地面凹凸不平，具有丘陵、洼地、缓坡等自然地形
- 坦克行驶在起伏地面上时，位置和姿态跟随地形变化（爬坡、下坡、倾斜）
- 障碍物从"砖墙/钢墙"变为岩石、沙袋掩体、废墟建筑、弹坑等野战元素
- 整体氛围从"城市巷战"转变为"旷野会战"

## 二、地形系统

### 2.1 高度图地形

使用 Three.js 的 `PlaneGeometry` 顶点位移实现高度图地形：

```
PlaneGeometry(mapWidth, mapDepth, segmentsX, segmentsZ)
```

- **地图尺寸**: 80 x 80 世界单位（比当前40x40大一倍，更开阔）
- **网格分辨率**: 80 x 80 段（每个世界单位1个顶点，足够平滑）
- **高度范围**: Y ∈ [-2, +4]（低洼到丘顶，总高差约6个单位）
- **生成方式**: 多层柏林噪声（Perlin Noise）叠加

### 2.2 噪声地形生成

使用多倍频噪声叠加，产生自然地貌：

| 倍频层 | 频率 | 振幅 | 作用 |
|--------|------|------|------|
| 第1层 | 0.02 | 3.0 | 大尺度丘陵起伏 |
| 第2层 | 0.05 | 1.5 | 中等坡丘细节 |
| 第3层 | 0.1 | 0.5 | 小尺度凹凸纹理 |

由于 Three.js 不自带噪声函数，需内嵌一个轻量 Simplex/Perlin 噪声实现（约50行代码）。

### 2.3 高度采样函数

提供 `getHeightAt(x, z)` 函数，给定世界坐标 XZ，返回该点地面高度 Y：

```javascript
getHeightAt(x, z) → number
```

实现方式：
1. 将世界坐标映射到地形网格的UV坐标
2. 找到对应的四个顶点（双线性插值）
3. 返回插值后的精确高度值

同时提供 `getNormalAt(x, z)` 函数，返回该点的地面法线向量，用于坦克倾斜。

### 2.4 特殊区域

在噪声地形基础上，叠加设计特殊区域：

| 区域类型 | 描述 | 实现方式 |
|----------|------|----------|
| **山脊** | 地图中央或侧翼的隆起地带 | 沿某方向施加正弦函数叠加 |
| **弹坑群** | 战场上的弹坑洼地 | 在随机位置叠加负向半球凹陷 |
| **平台高地** | 战术制高点 | 平滑抬升区域 + 平顶处理 |
| **河谷** | 低洼水域通道 | 沿路径压低地形 + 水面Mesh |
| **玩家基地区域** | 靠近基地的平坦区域 | 对基地周围高度做平滑过渡到0 |

### 2.5 地形材质

使用顶点颜色或多材质混合，根据高度和坡度着色：

| 高度/坡度 | 颜色 | 对应地貌 |
|-----------|------|----------|
| 低洼 (Y < 0) | 深棕 + 深绿 | 泥地/沼泽 |
| 平地 (Y ≈ 0) | 黄绿色 | 草地 |
| 缓坡 (中等高度) | 浅绿 + 黄色 | 草坡 |
| 高处 (Y > 2.5) | 灰色 + 棕色 | 岩石/裸土 |
| 陡坡 (法线倾斜大) | 深灰 | 岩壁 |

## 三、坦克物理适配

### 3.1 高度跟随

每帧更新坦克 Y 坐标：

```javascript
// 在 tank.update() 或 game.updatePlayerInput() 中
const groundY = terrain.getHeightAt(tank.x, tank.z);
tank.group.position.y = groundY;
```

### 3.2 坦克倾斜（俯仰和侧倾）

根据地形法线调整坦克旋转：

- **俯仰角（Pitch）**: 前进方向的坡度 → 绕局部X轴旋转
- **侧倾角（Roll）**: 垂直于前进方向的坡度 → 绕局部Z轴旋转

实现方式：
1. 采样坦克前后两点高度差 → 计算俯仰
2. 采样坦克左右两点高度差 → 计算侧倾
3. 使用 `lerp` 平滑过渡，避免抖动

```javascript
const frontY = getHeightAt(x + sin(rot) * 1.5, z + cos(rot) * 1.5);
const backY  = getHeightAt(x - sin(rot) * 1.5, z - cos(rot) * 1.5);
const pitch  = Math.atan2(frontY - backY, 3.0);

const leftY  = getHeightAt(x + cos(rot) * 1.0, z - sin(rot) * 1.0);
const rightY = getHeightAt(x - cos(rot) * 1.0, z + sin(rot) * 1.0);
const roll   = Math.atan2(leftY - rightY, 2.0);
```

### 3.3 坡度对速度的影响

```
实际速度 = 基础速度 × 坡度系数
```

| 坡度 | 速度系数 | 效果 |
|------|---------|------|
| 上坡（坡度 > 0.1） | 0.5 ~ 0.8 | 明显减速 |
| 平地（坡度 ≈ 0） | 1.0 | 正常速度 |
| 下坡（坡度 < -0.1） | 1.1 ~ 1.3 | 轻微加速 |
| 极陡坡（坡度 > 0.6） | 0 | 无法攀爬 |

坡度 = 坦克前方单位距离的高度变化率（即前进方向的切线斜率）。

### 3.4 碰撞盒适配

坦克碰撞盒的 Y 范围需跟随地形高度：

```javascript
updateBoundingBox() {
    const y = this.group.position.y;
    this.boundingBox.min.set(pos.x - s, y, pos.z - s);
    this.boundingBox.max.set(pos.x + s, y + 2, pos.z + s);
}
```


### 4.2 关卡差异

不再使用二维数组定义墙壁，改为**参数化地形生成** + **障碍物配置表**：

#### 关卡1 - 丘陵草原
- 地形：平缓丘陵，高差 ±2
- 障碍物：少量岩石散布，1-2个弹坑
- 敌人：4个普通坦克
- 特点：开阔视野，适合学习地形操控

#### 关卡2 - 河谷交锋
- 地形：中央河谷（低洼+水面），两侧高地
- 障碍物：河谷边沙袋掩体，两侧废墟建筑
- 敌人：3普通 + 2精英
- 特点：争夺河谷控制权，利用高低差射击

#### 关卡3 - 焦土战场
- 地形：剧烈起伏，密集弹坑，中央山脊
- 障碍物：大量弹坑、岩石群、废墟碎片
- 敌人：3普通 + 2精英 + 2重型
- 特点：复杂地形，需灵活利用掩体和地形优势

## 五、障碍物设计

所有障碍物均由基础几何图形拼装，保持项目风格。

### 5.1 岩石（不可摧毁）

- **功能**: 替代钢墙，不可摧毁的硬掩体
- **模型**: 多个不规则缩放的 `DodecahedronGeometry` 或变形的 `BoxGeometry` 堆叠
- **尺寸**: 大中小三档（大岩石可完全遮挡坦克）
- **颜色**: 灰色系，带苔藓绿色点缀
- **放置**: 自然散布，不对齐网格

### 5.2 沙袋掩体（可摧毁）

- **功能**: 替代砖墙，可被子弹击碎
- **模型**: 多个圆柱 `CylinderGeometry` 堆叠成弧形或直线
- **尺寸**: 高约1.2单位，长度可变（2-6个沙袋并排）
- **颜色**: 土黄色 + 深棕带子
- **特效**: 被击碎时散出沙粒粒子（黄色小方块）

### 5.3 废墟建筑

- **功能**: 部分掩体，内部可穿行
- **模型**: 不完整的墙壁（有缺口的BoxGeometry组合），破碎的地板
- **尺寸**: 5x5单位左右的废墟区
- **颜色**: 灰白色混凝土 + 暴露的砖红色
- **交互**: 不可摧毁，但提供视线遮蔽和掩护

### 5.4 弹坑

- **功能**: 天然掩体（坦克可驶入弹坑获得低姿态保护）
- **模型**: 地形高度局部凹陷（在噪声中叠加负高斯函数）
- **尺寸**: 直径3-5单位，深度1-2单位
- **视觉**: 坑底深棕色，坑边散落碎石

### 5.5 枯树/树桩

- **功能**: 视觉装饰 + 轻微遮挡
- **模型**: `CylinderGeometry` 树干 + 少量倾斜的 `BoxGeometry` 枯枝
- **尺寸**: 高2-3单位
- **交互**: 可被子弹/坦克撞断（可选）

### 5.6 基地防御工事

- **功能**: 玩家基地周围的保护结构
- **模型**: 低矮混凝土墙（BoxGeometry）+ 铁丝网（细线条）+ 沙袋
- **特点**: 基地区域地形平坦化处理，有进出通道

## 六、水域系统

### 6.1 河流/水塘

- 在地形低洼处（Y < 指定阈值）铺设水面 Mesh
- 水面为 `PlaneGeometry`，半透明蓝色材质
- 水面高度固定（如 Y = -0.5），地形低于水面的区域自然形成水域
- 坦克进入水域：大幅减速（50%速度），无法停留过久（可选：持续伤害）

### 6.2 浅滩与深水

| 水深 | 效果 |
|------|------|
| 浅滩（地形Y ∈ [-0.5, 0]） | 减速40%，可通行 |
| 深水（地形Y < -1.0） | 不可通行，坦克被阻挡 |

## 七、相机系统适配

### 7.1 高度跟随

```javascript
// 相机目标位置需加上坦克的地形高度
targetPos.y = cameraOffset.y + tankGroundY;

// 看向目标也需考虑高度
lookTarget.y = tankGroundY + 1;
```

### 7.2 防穿透

相机位置可能低于地形：
- 采样相机位置处的地形高度
- 如果 `camera.y < terrainY + 2`，则抬高相机
- 平滑过渡防止跳变

## 八、子弹系统适配

### 8.1 地形碰撞

子弹需检测是否击中地面：
```javascript
if (bullet.position.y < terrain.getHeightAt(bullet.x, bullet.z)) {
    // 子弹击中地面 → 创建尘土飞扬效果
    createDirtExplosion(bullet.position);
    bullet.destroy();
}
```

### 8.2 弹道物理（可选增强）

简单重力模拟：
```javascript
bullet.velocity.y -= gravity * deltaTime;  // gravity ≈ 3
bullet.position.add(bullet.velocity * deltaTime);
```

这使远距离射击时子弹微微下坠，增加射击技巧维度。

## 九、AI适配

### 9.1 路径规划

敌方AI需适应起伏地形：
- 避免尝试攀爬极陡坡（坡度 > 0.6）
- 优先选择平缓路线接近目标
- 利用弹坑和岩石作为掩体

### 9.2 射击调整

- AI需判断目标与自身高度差
- 如果目标在更高处且有地形遮挡，切换为绕行策略
- 利用高地优势向下射击

## 十、性能优化要点

| 优化项 | 策略 |
|--------|------|
| 地形网格 | 80x80段足够，不需要LOD |
| 障碍物 | 使用InstancedMesh做同类岩石 |
| 高度采样 | 缓存高度数据到Float32Array，避免每帧重建 |
| 阴影 | 地形接收阴影，方向光范围适当扩大 |
| 草丛装饰 | 仅在玩家附近生成（可选） |
| 物理更新 | 坦克倾斜用lerp平滑，每帧采样4个点即可 |
