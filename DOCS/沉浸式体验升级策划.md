## 一、专业 UI 界面 — 放大瞄准系统

### 1.1 功能描述

新增**放大瞄准模式**（Zoom / Scope），玩家可通过按键或鼠标右键切换进入放大视角，模拟坦克瞄准镜效果，提供更精确的远距离瞄准能力。

### 1.2 设计要点

| 项目 | 说明 |
|------|------|
| 触发方式 | 鼠标右键按住进入，松开退出（或按 `Z` 键切换） |
| 放大倍率 | 默认视角 FOV 60° → 放大后 FOV 30°（约 2 倍放大） |
| 过渡动画 | FOV 切换使用 `lerp` 平滑过渡，时长约 0.2 秒 |
| 瞄准镜 UI | 进入放大模式时，屏幕叠加半透明准星 HUD（十字线 + 距离刻度） |
| 画面效果 | 放大模式下可轻微降低相机跟随偏移，让视角更贴近炮管方向 |
| 移动限制 | 放大瞄准时坦克移动速度降低 50%，增加战术抉择感 |

### 1.3 涉及文件

- `js/game.js` — 相机控制逻辑（`updateCamera`、`cameraOffset`、FOV 调整）
- `js/game.js` — 输入监听（`setupInputListeners`，新增右键/Z 键监听）
- `index.html` / `css/style.css` — 新增准星 HUD 元素和样式

### 1.4 技术方案

```
// 伪代码示意
if (rightMouseDown || keys['KeyZ']) {
    targetFOV = 30;          // 放大
    moveSpeedMult = 0.5;     // 移动减速
    scopeOverlay.visible = true;
} else {
    targetFOV = 60;          // 恢复
    moveSpeedMult = 1.0;
    scopeOverlay.visible = false;
}
camera.fov = lerp(camera.fov, targetFOV, smooth);
camera.updateProjectionMatrix();
```

## 三、降低子弹发射频率

### 3.1 功能描述

## 四、后坐力系统

### 4.1 功能描述

发射子弹时，炮台产生**可见的后坐力动画**，在动画恢复前锁定射击，增强射击的物理反馈感。

### 4.2 设计要点

#### 4.2.1 后坐力动画

| 阶段 | 时长 | 表现 |
|------|------|------|
| **后坐** | 0.05~0.1 秒 | 炮管沿自身轴向**后缩约 0.3~0.5 单位**，炮塔轻微向上抬起约 2~3° |
| **恢复** | 0.3~0.5 秒 | 炮管平滑回到原位，炮塔角度归零 |
| **微震** | 后坐瞬间 | 整个坦克模型产生轻微随机抖动（幅度 ±0.02），持续 0.1 秒 |

#### 4.2.2 射击锁定机制

- 新增状态标志 `isRecoiling`，发射时设为 `true`
- 后坐力动画**完全恢复后**才将 `isRecoiling` 设为 `false`
- `canShoot()` 方法需同时检查 `shootTimer <= 0` **且** `!isRecoiling`
- 实际射击间隔 = max(shootCooldown, 后坐力恢复时间)

#### 4.2.3 动画实现思路

```
// Tank 类新增属性
this.isRecoiling = false;
this.recoilOffset = 0;       // 炮管后缩偏移量
this.recoilPitch = 0;        // 炮塔仰角偏移
this.recoilPhase = 'idle';   // idle | recoil | recover

// 发射时触发
startRecoil() {
    this.isRecoiling = true;
    this.recoilPhase = 'recoil';
    this.recoilOffset = 0;
    this.recoilPitch = 0;
}

// 每帧更新
updateRecoil(deltaTime) {
    if (this.recoilPhase === 'recoil') {
        this.recoilOffset = lerp(this.recoilOffset, -0.4, deltaTime * 30);
        this.recoilPitch = lerp(this.recoilPitch, 0.05, deltaTime * 30);
        if (this.recoilOffset <= -0.35) this.recoilPhase = 'recover';
    } else if (this.recoilPhase === 'recover') {
        this.recoilOffset = lerp(this.recoilOffset, 0, deltaTime * 3);
        this.recoilPitch = lerp(this.recoilPitch, 0, deltaTime * 3);
        if (Math.abs(this.recoilOffset) < 0.01) {
            this.recoilOffset = 0;
            this.recoilPitch = 0;
            this.recoilPhase = 'idle';
            this.isRecoiling = false;
        }
    }
    // 应用到炮管模型的 position.z 和 rotation.x
}
```

### 4.3 涉及文件

- `js/tank.js` — 新增 `startRecoil()`、`updateRecoil()`、修改 `canShoot()`
- `js/game.js` — 射击逻辑中调用 `startRecoil()`
- `js/ai.js` — 敌方坦克射击时同样触发后坐力（可选）

## 五、炮弹外观重做

### 5.1 功能描述

将当前的黄色发光球体子弹，改为接近真实坦克炮弹的造型，并增加飞行拖尾效果。

### 5.2 当前状态

- 球体 `SphereGeometry(0.15)`，黄色发光材质
- 附带点光源
- 无拖尾效果

### 5.3 新设计方案

#### 5.3.1 炮弹模型

| 部件 | 几何体 | 说明 |
|------|--------|------|
| 弹头 | `ConeGeometry(0.08, 0.2, 8)` | 锥形弹头，深灰色金属质感 |
| 弹体 | `CylinderGeometry(0.08, 0.08, 0.3, 8)` | 圆柱弹体，铜色/深灰色 |
| 弹尾 | `CylinderGeometry(0.08, 0.06, 0.1, 8)` | 略收窄的尾部 |

整体组合为一个 `Group`，沿飞行方向旋转对齐。

#### 5.3.2 飞行拖尾

- 使用简易的**粒子拖尾**：每帧在炮弹尾部生成 1~2 个半透明小粒子
- 粒子颜色：橙黄色 → 灰色渐变，模拟火药燃气
- 粒子生命周期约 0.2~0.3 秒，逐渐缩小并变透明
- 保持性能：每颗炮弹最多同时存在 10~15 个拖尾粒子

#### 5.3.3 飞行姿态

- 炮弹模型始终朝飞行方向对齐（使用 `lookAt` 或手动设置 rotation）
- 可添加轻微自旋（绕飞行轴），模拟膛线旋转效果

### 5.4 涉及文件

- `js/bullet.js` — 重写 `Bullet` 类的模型构建和 `update` 逻辑

## 六、击中特效差异化

### 6.1 功能描述

根据子弹击中的不同目标材质，产生差异化的视觉特效，大幅提升战斗反馈的层次感。

### 6.2 特效设计矩阵

| 碰撞类型 | 爆炸 | 烟雾 | 火花 | 碎屑 | 附加效果 |
|----------|:----:|:----:|:----:|:----:|----------|
| **坦克** | 中型爆炸 | 黑烟柱 | 金属火星四溅 | 装甲碎片 | 被击坦克闪红 + 击杀时大爆炸 |
| **砖墙** | 小型爆炸 | 土黄尘雾 | — | 砖块碎屑飞溅 | 棕红色碎块粒子 |
| **钢墙** | 无爆炸 | 少量烟雾 | 明亮金属火花 | — | 火花飞溅 + 弹跳音效 |
| **基地** | 大型爆炸 | 浓烟 | 火星 | 建筑碎片 | 火焰持续燃烧效果（短暂） |
| **边界/出界** | 无 | — | — | — | 子弹自然消失 |

### 6.3 特效实现方案

#### 6.3.1 烟雾系统（新增）

```
createSmoke(position, color, count) {
    // 使用半透明球体模拟烟雾粒子
    // 颜色：黑烟(0x222222)、土烟(0x8B7355)、白烟(0xAAAAAA)
    // 行为：向上缓慢飘动，逐渐扩大并变透明
    // 生命周期：0.5~1.5 秒
}
```

#### 6.3.2 火花系统（新增）

```
createSparks(position, count, color) {
    // 使用极小的亮色粒子（0.03~0.05 大小）
    // 颜色：金属火花(0xFFAA00/0xFFFFAA)
    // 行为：高速四散飞射，受重力影响呈抛物线
    // 生命周期：0.3~0.6 秒
    // 可附带微型点光源增强视觉效果
}
```

#### 6.3.3 碎屑系统（新增）

```
createDebris(position, material, count) {
    // 根据 material 类型决定碎屑颜色和形状
    // 'brick': 棕红色小方块，0.05~0.1 大小
    // 'metal': 深灰色薄片，0.03~0.08 大小
    // 'concrete': 灰白色不规则块，0.05~0.12 大小
    // 行为：高速弹射 + 重力下落 + 旋转
    // 生命周期：0.5~1.0 秒
}
```

#### 6.3.4 各碰撞类型的特效组合调用

```
// 击中坦克
createExplosion(pos, 'medium');
createSmoke(pos, 0x222222, 8);
createSparks(pos, 20, 0xFFAA00);     // 金属火星四溅
createDebris(pos, 'metal', 6);

// 击中砖墙
createExplosion(pos, 'small');
createSmoke(pos, 0x8B7355, 5);       // 土黄尘雾
createDebris(pos, 'brick', 10);

// 击中钢墙
createSparks(pos, 30, 0xFFFFAA);     // 明亮金属火花
createSmoke(pos, 0xAAAAAA, 3);

// 击中基地
createExplosion(pos, 'large');
createSmoke(pos, 0x111111, 12);      // 浓烟
createSparks(pos, 15, 0xFF6600);
createDebris(pos, 'concrete', 8);
```

### 6.4 涉及文件

- `js/effects.js` — 新增 `createSmoke()`、`createSparks()`、`createDebris()` 方法
- `js/collision.js` — 在各碰撞分支中调用对应特效组合
- `js/game.js` — 碰撞结果处理中补充特效调用

## 九、验证方案

1. **放大瞄准**：右键按住确认 FOV 平滑缩小，松开恢复；准星 HUD 正确显示/隐藏
2. **激光瞄准**：确认游戏中无绿色虚线和红色弹着点
3. **发射频率**：连续按空格，确认发射间隔明显增大
4. **后坐力**：发射后观察炮管后缩动画，确认动画恢复前无法再次射击
5. **炮弹外观**：观察飞行中的炮弹呈锥柱造型，带拖尾效果
6. **击中特效**：分别击中坦克/砖墙/钢墙/基地，确认各自特效差异明显
