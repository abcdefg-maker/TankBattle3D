# TankBattle3DD - 游戏生成提示词

## 核心提示词

```
请使用Three.js开发一个3D版坦克大战游戏，具体要求如下：

### 基本要求
1. 使用Three.js作为3D渲染引擎，通过CDN引入，无需任何构建工具
2. 纯前端实现，打开index.html即可直接在浏览器中运行
3. 采用第三人称视角（相机在玩家坦克后上方跟随）
4. 所有3D模型全部由基本几何图形拼凑而成（BoxGeometry、CylinderGeometry、SphereGeometry等）

### 坦克模型设计
- 车身：扁平长方体（主体），军绿色
- 炮塔：较小的立方体，放在车身上方
- 炮管：细长圆柱体，从炮塔前方伸出
- 履带：车身两侧各一个深色扁长方体
- 敌方坦克结构相同，使用灰色/红色区分类型

### 操作控制
- WASD或方向键控制坦克前进/后退/左转/右转
- 空格键发射子弹
- 鼠标控制炮塔旋转方向
- ESC键暂停游戏

### 地图系统
- 20x20的网格地图
- 砖墙：棕红色立方体，可被子弹摧毁
- 钢墙：银灰色立方体，不可摧毁
- 水域：蓝色半透明平面，阻挡坦克通行
- 草丛：绿色半透明平面，可通行但遮挡视线
- 基地：位于地图底部中央，被摧毁则游戏失败

### 子弹系统
- 子弹为小型球体，黄色带发光效果
- 直线飞行，碰到墙壁或坦克后消失并触发效果
- 玩家子弹击中砖墙：砖墙消失
- 玩家子弹击中钢墙：子弹消失，墙不变
- 玩家子弹击中敌人：敌人爆炸
- 敌人子弹击中玩家：玩家扣血

### 敌方AI
- 巡逻模式：随机方向移动
- 追踪模式：感知到玩家时朝玩家移动
- 攻击模式：面向玩家时自动开火
- 50%概率朝基地方向行进

### 关卡设计
- 3个关卡，难度递增
- 关卡1：4个普通敌人，简单地形
- 关卡2：5个敌人（含精英），复杂巷道
- 关卡3：7个敌人（含重型），混合地形

### 视觉效果
- 环境光 + 方向光照明系统
- 爆炸粒子效果
- 开火闪光效果
- 地面网格纹理

### UI系统
- 开始菜单界面（含标题和开始按钮）
- 游戏HUD：生命值、分数、剩余敌人、当前关卡
- 暂停菜单
- 游戏结束/胜利界面

### 道具系统
- 击杀敌人有概率掉落道具
- 加速道具（星星）：提升移速5秒
- 护盾道具（盾牌）：无敌3秒
- 炸弹道具：清除所有当前敌人
```

## 分模块提示词

### 模块1：场景初始化
```
使用Three.js创建基础游戏场景：
- 初始化Scene、PerspectiveCamera、WebGLRenderer
- 渲染器尺寸为窗口大小，开启抗锯齿和阴影
- 添加环境光（AmbientLight，强度0.6）和方向光（DirectionalLight，强度0.8，开启阴影）
- 创建深绿色地面平面（PlaneGeometry）作为游戏区域
- 在地面上绘制网格线辅助定位
- 设置天空色背景
- 建立requestAnimationFrame渲染循环
- 使用时钟（Clock）计算deltaTime实现帧率无关的运动
```

### 模块2：坦克模型
```
用Three.js基础几何体创建坦克模型：
- 使用Group将各部件组合
- 车身：BoxGeometry(2, 0.6, 3)，军绿色MeshStandardMaterial
- 炮塔：BoxGeometry(1.2, 0.5, 1.2)，放在车身上方，可独立旋转
- 炮管：CylinderGeometry(0.1, 0.1, 2)，水平放置从炮塔前伸出
- 左履带：BoxGeometry(0.4, 0.4, 3)，深色，放在车身左侧
- 右履带：BoxGeometry(0.4, 0.4, 3)，深色，放在车身右侧
- 坦克整体可旋转和移动
- 炮塔可独立于车身旋转（鼠标控制）
```

### 模块3：第三人称相机
```
实现第三人称跟随相机：
- 相机位于坦克后方偏上位置（相对坦克：后方距离8，高度6）
- 相机始终看向坦克位置（略高于坦克中心）
- 使用lerp实现平滑跟随效果
- 坦克转向时相机跟随旋转
- 确保相机不穿透墙壁（可简化处理）
```

### 模块4：控制系统
```
实现坦克键盘控制：
- 监听keydown和keyup事件，维护按键状态表
- W/上：沿坦克前方向移动
- S/下：沿坦克后方向移动
- A/左：坦克左转（旋转Y轴）
- D/右：坦克右转
- 空格：发射子弹（有冷却时间0.5秒）
- 移动速度：5单位/秒，旋转速度：3弧度/秒
- 鼠标移动控制炮塔旋转（Raycaster投射到地面计算角度）
```

### 模块5：地图系统
```
实现网格地图系统：
- 20x20的二维数组表示地图
- 0=空地, 1=砖墙, 2=钢墙, 3=水域, 4=草丛, 5=基地
- 每格大小为2x2单位
- 砖墙：BoxGeometry(2,2,2)棕红色，可被摧毁
- 钢墙：BoxGeometry(2,2,2)银灰色金属材质
- 水域：PlaneGeometry(2,2)蓝色透明材质，y位置略高于地面
- 草丛：PlaneGeometry(2,2)绿色透明材质，y位置高于坦克
- 基地：特殊几何体组合，位于地图底部中央
- 提供3个关卡的地图数据
```

### 模块6：碰撞检测
```
实现AABB碰撞检测系统：
- 每个游戏实体都有一个AABB包围盒（Box3）
- 坦克移动前先检测目标位置是否会与墙壁碰撞
- 如果碰撞则阻止移动
- 子弹每帧检测是否击中坦克或墙壁
- 坦克与坦克之间也要检测碰撞
- 坦克不能移动到水域上
- 坦克不能超出地图边界
```

### 模块7：子弹系统
```
实现子弹发射和飞行：
- 子弹从炮管前端生成
- SphereGeometry(0.15)黄色带emissive发光
- 沿炮管朝向直线飞行，速度15单位/秒
- 设置最大飞行距离，超过后自动消失
- 碰撞处理：
  - 击中砖墙→墙消失+子弹消失+爆炸效果
  - 击中钢墙→子弹消失
  - 击中敌方坦克→坦克爆炸+子弹消失+加分
  - 击中玩家坦克→扣血+子弹消失
  - 击中基地→游戏失败
```

### 模块8：敌方AI
```
实现敌方坦克AI行为：
- 状态机模式：巡逻/追踪/攻击
- 巡逻：每2-4秒随机选择一个方向移动
- 追踪：当玩家距离<15单位时，计算朝向玩家的方向移动
- 攻击：面朝玩家方向（角度差<30度）时以1-2秒间隔开火
- 遇到障碍物时随机改变方向
- 敌人从地图顶部3个出生点生成
- 同时最多4个敌人在场
- 生成间隔3秒
- 三种类型：普通（灰色/1HP）、精英（红色/1HP/快速）、重型（深红/2HP）
```

### 模块9：特效系统
```
实现爆炸和视觉特效：
- 爆炸效果：
  - 生成20-30个小型BoxGeometry碎片
  - 碎片向四周随机飞散
  - 颜色从黄→橙→红渐变
  - 碎片逐渐缩小并淡出
  - 1秒后移除所有碎片
- 开火闪光：
  - 炮口处短暂出现白色PointLight
  - 持续0.1秒后消失
- 可选：坦克移动时地面灰尘粒子
```

### 模块10：UI系统
```
使用HTML/CSS实现游戏UI覆盖层：
- 开始界面：
  - 游戏标题"TANK BATTLE 3D"大字
  - "开始游戏"按钮
  - 简单操作说明
- 游戏HUD（position:fixed覆盖在canvas上）：
  - 左上：红心图标x生命数
  - 右上：分数显示
  - 右侧：剩余敌人图标
  - 底部：关卡信息
- 暂停界面：半透明遮罩+"继续"/"重新开始"按钮
- 结束界面：显示最终分数和"重新开始"按钮
```

## 性能优化提示词
```
对3D坦克大战进行性能优化：
- 对静态墙壁使用InstancedMesh减少draw call
- 及时dispose不再使用的几何体和材质
- 子弹和粒子使用对象池模式
- 限制同屏粒子数量上限
- 使用frustum culling（Three.js默认开启）
- 限制阴影贴图分辨率
- 对远处物体降低细节
```

## 调试和测试提示词
```
为游戏添加调试功能：
- 按F1切换线框模式
- 按F2显示/隐藏碰撞盒
- 按F3显示FPS计数器
- 控制台输出关键游戏状态信息
- 确保窗口resize时正确更新渲染器和相机
```
